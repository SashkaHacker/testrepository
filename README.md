# Готовый проект после урока 16

Это состояние Recipe Manager API **после выполнения методички урока 16**. В проекте есть:

- регистрация, логин и JWT;
- `GET /users/me`;
- связь `User → Recipe` через `owner_id`;
- Alembic-миграция для добавления владельца;
- CRUD рецептов с проверкой прав владельца.

## Правила доступа

- `owner_id` не передаётся клиентом в JSON: сервер подставляет `current_user.id` из JWT;
- `GET /recipes` возвращает только рецепты текущего пользователя;
- `GET`, `PATCH` и `DELETE /recipes/{recipe_id}` проверяют владельца;
- чужой рецепт существует, но для другого пользователя API отвечает `403`.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload
```

Swagger: <http://127.0.0.1:8000/docs>

## Точка проверки перед уроком

1. Зарегистрировать пользователя через `POST /register`.
2. Получить JWT через `POST /login` (`username` и `password` передаются как form data).
3. Авторизоваться через кнопку **Authorize** в Swagger.
4. Проверить `GET /users/me`.
5. Создать рецепт через `/recipes`.
6. Зарегистрировать второго пользователя и убедиться, что он не может получить, изменить или удалить рецепт первого (`403`).

## Миграция и учебная SQLite-база

Если в старой SQLite-базе уже есть рецепты без владельца, удалите учебный файл `recipes.db` и примените миграции заново. Новая обязательная колонка `owner_id` не может быть автоматически заполнена для старых рецептов: база не знает их владельцев.
